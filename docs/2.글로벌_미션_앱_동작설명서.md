# 글로벌 미션 앱 동작 설명서

## 문서 목적
본 문서는 글로벌 미션 앱의 각 화면에 대해 아래 내용을 명확히 정의한다.

- 화면의 목적
- 화면 내 섹션 구성
- 클릭 가능한 요소
- 각 요소 클릭 시 동작
- 상태별 UI 변화
- 다음 화면 또는 모달/토스트 이동 규칙

이 문서는 디자인 참고 이미지와 함께 사용되며,  
이 문서를 기준으로 화면 구조 / 라우팅 / 상태 처리 / 모달 / 토스트 / 버튼 활성 조건을 구현해야 한다.

---

## 공통 규칙

### 0. 제품명·내비게이션·알림
- 앱 표시 이름: **Localy**
- 하단 탭(순서): **Home · Board · World · My** (문서 기준이며, 과거 목업의 「Feed」 표기는 **Board**로 통일. Feed라는 탭·용어는 사용하지 않음)
- 알림: 하단 탭에 두지 않고, **홈(및 앱 공통 상단) 우측 벨 아이콘**으로 진입

### 1. 용어 통일
아래 용어는 전 문서에서 동일한 의미로 사용한다.

- **홈**: 앱 최초 진입 후 메인 화면(네비게이션바 첫 번째 아이콘에 해당하는 화면)
- **오늘의 팀 미션**: 오늘의 미션 상세 + 팀 상태 + 진행 상태를 함께 보여주는 화면
- **미션 참여**: 사진 업로드 및 한 줄 작성 화면
- **결과 보기**: 팀 결과 카드 상세 화면
- **Board**: 다른 팀의 결과 카드 목록 화면(네비게이션바 두 번째 아이콘에 해당하는 화면)
- **World**: 전 세계 참여 현황 요약 화면(네비게이션바 세 번째 아이콘에 해당하는 화면)
- **My**: 프로필 / 활동 / 설정 화면(네비게이션바 네 번째 아이콘에 해당하는 화면)
- **결과 카드**: 두 명의 제출물을 묶은 카드 단위(참고: Board에 노출되는 카드)
- **표현 반응**: 선택형 한 줄 반응
- **공통 반응**: 아이콘형 반응

### 2. 버튼 상태 규칙
버튼은 아래 세 상태 중 하나를 가진다.

- **활성(active)**: 클릭 가능
- **비활성(disabled)**: 클릭 불가
- **로딩(loading)**: 진행 중 상태, 클릭 불가

### 3. 피드백 방식 규칙

#### 모달
유저가 반드시 확인/선택해야 하는 상황

#### 바텀시트
짧은 선택을 해야 하나 별도 페이지는 불필요한 상황

#### 토스트
단순 성공/완료 안내

### 4. MVP 모달 / 토스트 규칙

#### 모달
- 매칭 시작 확인
- 작성 중 이탈 확인
- 로그아웃 확인
- 업로드 실패 / 매칭 실패 재시도

#### 바텀시트
- 표현 남기기 선택

#### 토스트
- 미션 제출 완료
- 저장 완료
- 저장 해제 완료
- 표현 선택 완료
- 공유 / 복사 완료
- 아이콘 반응 완료(선택)

### 5. 알림 이벤트 규칙
MVP 알림 이벤트는 아래 4개로 고정한다.

- 매칭 완료
- 상대 제출 완료 → 결과 보기 가능
- 새 반응 도착(묶음형 추천)
- 오늘의 새 미션 오픈(선택사항 / 후순위)

# 화면 정의

## 화면 1. 이메일 가입 / 로그인 화면

### 목적
사용자가 가장 낮은 진입장벽으로 가입 또는 로그인하도록 한다.

### 주요 구성
- 앱 로고 / 서비스 소개
- 이메일 입력 필드
- 인증 / 다음 버튼
- 로그인 전환 또는 가입 안내 문구

### 클릭 요소 및 동작

#### 이메일 입력
- 유효한 이메일 형식 검사
- 형식이 틀리면 다음 버튼 비활성

#### 다음 버튼
- 이메일 유효 시 활성
- 클릭 시 가입 / 로그인 처리 시작
- 성공 시 최초 접속 여부 판단

### 이동 규칙
- **최초 접속 사용자** → 프로필 / 관심사 설정 화면으로 강제 이동
- **기존 사용자** → 홈으로 이동

---

## 화면 2. 최초 프로필 / 관심사 설정 화면

### 목적
매칭에 필요한 최소 프로필과 관심사를 입력받는다.  
이 화면은 가입 자체가 아니라 앱 시작 전 필수 초기 설정 화면이다.

### 주요 구성

#### 섹션 1. 프로필 사진
- 프로필 이미지 영역
- 사진 변경 버튼 또는 기본 이미지 사용

#### 섹션 2. 기본 정보
- 닉네임
- 나이
- 국가
- 한 줄 프로필

#### 섹션 3. 관심사
- 음식
- 일상
- 장소
- 감정
- 공부
- 일상소비

#### 하단 버튼
- 시작하기

### 클릭 요소 및 동작

#### 프로필 사진 변경
- 필수 입력 요소 아님
- 이미지 선택 UI 오픈
- 이미지 선택 후 프리뷰 갱신

#### 닉네임 입력
- 2~12자
- 조건 미충족 시 저장 불가

#### 나이 선택
- 드롭다운 / 선택형

#### 국가 선택
- 드롭다운 / 선택형
- 이후 매칭 기준에 사용

#### 한 줄 프로필 입력
- MVP **최대 40자**
- 예: `32세 · Founder`

#### 관심사 칩 클릭
- 다중 선택 가능
- 최소 1개 이상 선택 필요

#### 시작하기 버튼
- 필수 항목 모두 충족 시 활성
- 클릭 시 저장 후 홈 이동

### 상태 규칙
- 필수 항목 미입력 시 시작하기 비활성
- 저장 중에는 버튼 로딩

## 화면 3. 홈 화면

### 목적
사용자에게 오늘의 행동 시작점과 Board 진입점을 제공한다.

### 주요 구성

#### 섹션 1. 오늘의 미션
포함 요소:
- 미션 카테고리
- 마감 정보
- 미션 제목
- 짧은 설명
- 매칭 상태 박스
- 상태 기반 버튼

#### 섹션 2. Board
포함 요소:
- 다른 팀 카드 일부 그리드
- Board 전체 보기 버튼

#### Board 정렬·필터 (MVP)
- Board(및 필요 시 홈의 Board 미리보기)에서 **정렬**만 둔다.
- MVP 옵션: **최신순** · **인기순** (API: `GET /board/cards?sort=latest|popular`, 상세는 API 정의서)
- 추후 국가·카테고리 등 필터는 같은 화면에서 **확장 가능**하게 설계

### 오늘의 미션 섹션 상태 정의

#### 상태 A. 매칭 전
- 매칭 상태 문구 표시
- 버튼명: `매칭 시작`
- 버튼 활성

#### 상태 B. 매칭 중
- 매칭 상태 문구 표시
- 버튼명: `매칭 중...`
- 버튼 비활성

#### 상태 C. 매칭 완료
- 상대 국가 표시
- 버튼명: `오늘의 미션 참여`
- 버튼 활성

### 클릭 요소 및 동작

#### 오늘의 미션 > 매칭 시작
- 매칭 시작 확인 모달 오픈

#### 오늘의 미션 > 매칭 중...
- 클릭 불가

#### 오늘의 미션 > 오늘의 미션 참여
- 오늘의 팀 미션 화면으로 이동

#### Board 카드 미리보기 클릭
- 해당 카드의 결과 보기 화면으로 이동

#### Board 전체 보기 클릭
- Board 화면으로 이동

### 모달

#### 매칭 시작 확인 모달 (최종 카피 고정)
- 제목: `오늘의 팀을 찾을까요?`
- 본문: `앱을 닫아도 매칭되면 알려드려요.`
- 버튼: `취소` / `시작하기`

동작:
- 시작하기 클릭 시 매칭 요청 생성
- 홈 오늘의 미션 섹션 상태를 `매칭 중`으로 변경

## 화면 4. 오늘의 팀 미션 화면

### 목적
오늘의 미션 정보, 팀 정보, 진행 상태를 한 화면에서 보여주고  
다음 행동(미션하기 또는 결과 보기)을 실행하게 한다.

### 주요 구성

#### 섹션 1. 오늘의 미션
- 카테고리 배지
- 마감 정보
- 미션 제목
- 짧은 설명
- 주의사항 i 아이콘

#### 섹션 2. 내 팀 정보
- 나 카드
  - 국기
  - 닉네임
  - 짧은 프로필
- 상대 카드
  - 국기
  - 닉네임
  - 짧은 프로필

#### 섹션 3. 진행 상태
- 진행률 바
- 나 상태
- 상대 상태
- 상태 기반 메인 버튼
- 상태 안내 문구

### 주의사항 아이콘 동작
i 아이콘 클릭 시 작은 말풍선 / 툴팁 표시

내용:
- 개인정보가 드러나는 사진은 피해주세요
- 타인의 얼굴이 선명한 사진은 피해주세요
- 민감한 위치 정보는 피해주세요

### 진행 상태별 규칙 (팀 미션 상태 4분기와 대응)

#### `not_started`
- 진행률 `0/2`
- 버튼명: `미션하기`
- 버튼 활성
- 클릭 시 미션 참여 화면 이동

#### `peer_submitted_waiting` (상대만 제출 완료)
- 진행률 `1/2`
- 버튼명: `미션하기`
- 버튼 활성
- 보조 문구: `상대 팀원은 이미 답변을 완료했어요`
- 클릭 시 미션 참여 화면 이동

#### `self_submitted_waiting` (나만 제출 완료)
- 진행률 `1/2`
- 버튼명: `결과 보기`
- 버튼 비활성
- 안내 문구:
  - 상대 팀원의 업로드가 완료되면 결과 보기가 활성화됩니다

#### `ready_to_view`
- 진행률 `2/2`
- 버튼명: `결과 보기`
- 버튼 활성
- 클릭 시 결과 보기 화면 이동

### 클릭 요소 및 동작

#### 뒤로가기
- 홈으로 이동

#### 미션하기 버튼
- 미션 참여 화면 이동(제출 완료 후에는 이 버튼·경로로 미션 참여 화면 재진입 불가. 아래 「제출 후 네비게이션」 참고)

#### 결과 보기 버튼(비활성)
- 동작 없음

#### 결과 보기 버튼(활성)
- 결과 보기 화면 이동

### 제출 후 네비게이션 (MVP 고정)
- 당일 해당 미션·팀에 대해 **한 번 제출이 완료되면**, 사용자는 **미션 참여(입력) 화면으로 다시 진입할 수 없다**
- 제출 후에는 **오늘의 팀 미션 화면**으로만 복귀하며, 화면에는 `self_submitted_waiting` 또는 `ready_to_view` 상태만 표시된다(별도 읽기 전용 요약 화면은 두지 않음)
- 딥링크·뒤로가기 등으로 미션 참여 경로가 열리지 않도록 라우팅에서 차단

## 화면 5. 미션 참여 화면

### 목적
유저가 사진 1장과 한 줄 설명을 제출하는 실행 전용 화면이다.

### 주요 구성

#### 섹션 1. 미션 요약
- 카테고리
- 미션 제목
- 짧은 설명 1줄

#### 섹션 2. 사진 업로드
- 업로드 박스
- 선택 후 이미지 프리뷰
- 삭제 / 다른 사진 선택 버튼
- MVP: 허용 **jpg / jpeg / png**, **HEIC**는 가능하면 **JPEG 변환 후** 업로드 — 업로드 전 **압축**(긴 변 1600px 이하·2MB 이하 권장, 서비스 정의서 `1-13-1` 참고)

#### 섹션 3. 한 줄 설명 입력
- 텍스트 입력창
- 글자 수 표시
- 주의사항 i 아이콘

#### 하단 버튼
- 제출하기

### 클릭 요소 및 동작

#### 사진 업로드 박스 클릭
- 이미지 선택
- 성공 시 프리뷰 노출

#### 삭제 클릭
- 선택된 이미지 제거
- 업로드 박스 상태로 복귀

#### 한 줄 설명 입력
- 글자 수 실시간 카운트
- MVP **최대 60자**

#### 주의사항 아이콘 클릭
- 말풍선 / 툴팁 표시

#### 제출하기 버튼
조건:
- 사진 존재
- 한 줄 설명 존재

조건 충족 시 활성  
클릭 시 제출 요청

### 제출 성공 시 동작
- 토스트: `제출이 완료되었어요`
- 오늘의 팀 미션 화면으로 복귀
- 내 상태를 업로드 완료로 갱신

### 예외

#### 작성 중 뒤로가기
- 작성 중 이탈 확인 모달 노출

#### 업로드 실패
- 업로드 실패 모달 노출

### 제출 완료 후 재진입
- 해당 일·해당 팀 미션에 대해 제출이 이미 완료된 경우, 이 화면으로의 진입은 허용하지 않음(오늘의 팀 미션 화면에 머무름)

## 화면 6. 결과 보기 화면

### 목적
두 국가 팀의 제출물을 비교하고, 반응하고, 저장하거나 공유하는 카드 소비 화면이다.

### 공통 성격
이 화면은 아래 두 경우에 동일한 구조로 사용한다.

- 내 팀 결과 보기
- Board에서 본 다른 팀 결과 보기

즉 결과 카드 상세 화면은 하나이며, 맥락만 다르다.

### 주요 구성

#### 섹션 1. 상단 헤더
- 뒤로가기
- 제목: `결과 보기`

#### 섹션 2. 미션 정보
- 카테고리 배지
- 미션 제목

#### 섹션 3. 팀 결과 카드
- 한 줄 비교 문구
- 유저 A 카드
  - 국기
  - 닉네임
  - 짧은 프로필
  - 이미지
  - 캡션
- 유저 B 카드
  - 국기
  - 닉네임
  - 짧은 프로필
  - 이미지
  - 캡션

#### 섹션 4. 빠른 반응
- 공통 아이콘 반응 4개
- 숫자 누적 표시

#### 섹션 5. 표현 남기기
- 표현 총 개수 표시
- 표현 선택 버튼
- 자주 선택된 표현 목록

#### 섹션 6. 보조 액션
- 공유하기
- (선택) 저장 아이콘

### 빠른 반응 동작

#### 아이콘 반응 클릭
- **다른 아이콘**을 누르면 기존 선택이 **새 아이콘으로 변경**되고 집계가 갱신된다
- **같은 아이콘**을 다시 누르면 반응이 **취소**되고 집계가 갱신된다
- 필요 시 토스트 또는 숫자 즉시 반영

### 표현 선택 동작

#### 표현 선택 클릭
- 표현 선택 바텀시트 오픈

바텀시트 내용:
- 카테고리에 맞는 표현 목록
- 항목 탭은 로컬 선택 상태만 갱신(서버 저장 없음)
- 사용자가 **`선택 완료`** 버튼을 눌렀을 때만 `POST`로 서버 반영

선택 완료(서버 반영) 후:
- 표현 수 갱신
- 표현 목록 갱신
- 토스트: `표현을 남겼어요`

### 저장 동작(있을 경우)

#### 저장 클릭
- 카드 저장
- 토스트: `카드가 저장되었어요`

#### 저장 해제 클릭
- 저장 해제
- 토스트: `저장이 해제되었어요`

### 공유하기 클릭 (MVP)
- **링크 공유 우선**(결과 카드 상세로 이어지는 URL)
- 권장 형식: `app://result-card/{resultCardId}` 및 `https://your-domain/result-card/{resultCardId}` (`your-domain`은 출시 도메인 확정 시 반영)
- 이미지 공유·OG 카드 등은 **후순위**
- 토스트: `링크가 복사되었어요` 또는 공유 성공 안내

# 상태 화면 / 예외 처리

## A. 매칭 중 상태
**위치:** 홈 오늘의 미션 카드 내부

**표시:**
- 버튼 `매칭 중...` 비활성
- 문구:
  - `비슷한 시간대의 다른 국가 팀원을 찾고 있어요`

## B. 제출 완료 상태
**위치:** 오늘의 팀 미션 화면 내부

**표시:**
- 진행률 `1/2`
- 상대 대기 문구
- 결과 보기 비활성

## C. 결과 대기 상태
**위치:** 오늘의 팀 미션 화면 내부

**표시:**
- 결과 보기 비활성
- 안내 문구 유지

## D. 빈 상태
**필수 대상:**
- 내 결과 카드 목록
- 저장한 카드 목록
- Board
- World

각 빈 상태에는 반드시 아래 요소가 있어야 한다.

- 짧은 설명
- 다음 행동 버튼

## E. 실패 상태
**필수 대상:**
- 매칭 실패
- 업로드 실패
- 저장 실패
- 반응 실패

**표현 방식:**
- 모달 또는 토스트 + 재시도 버튼
